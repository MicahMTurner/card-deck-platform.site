Quick Start :

when creating a new game, beofre writing your own logic you need to create the following classes:

  • a class that extends Game class
  • player handler that extends PlayerEventsHandler
  • public handler that extends PublicEventsHandler
  • card handler that extends CardEventsHandler
That's it! easy as 1,2,3.. now you are ready to fill in your logic in the handlers. for more advanced options please read the JavaDoc and see our war game tutorial at "Create Plugin Tutorial" tab.

Since android has it's own VM, before you can use your exported ".jar" file you need to convert it to android's format (Dalvik VM).
for that you need to perform the following steps:
open the command line/terminal and browse to the following path: Android\android-sdk\platform-tools.
there insert the following command:
dx --dex --keep-classes --output= <path+destination> <path+src>.
for example:
dx --dex --keep-classes --output= c:\converted\war.jar c:\myGame\war.jar

when creating a game, make sure to add an xml file named "info.xml" (case sensitive), under "src" dir, with the following format:

      <?xml version="1.0" encoding="UTF-8"?>       <Game>
            <classPath>war.War</classPath>
      </Game>

where classPath is the path of the class that extends "Game".
in this example we got a package named war and in it we have the "War" class that extends "Game".
